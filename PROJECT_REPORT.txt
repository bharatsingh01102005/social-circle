                           PAGE 1: EXECUTIVE SUMMARY

This report presents a comprehensive overview of a Social Media Web Application 
developed as a mini project. The application is a full-stack solution built 
using the MERN stack (MongoDB, Express.js, React.js, and Node.js), designed 
to provide users with a modern social networking experience.

The project implements core social media functionalities including user 
authentication, post creation and management, real-time notifications, story 
sharing, messaging capabilities, and social interactions such as likes, 
comments, and following other users. The application features a responsive 
user interface built with React and Tailwind CSS, coupled with a robust 
RESTful API backend powered by Node.js and Express.

Key highlights of the project include:
- Secure user authentication with JWT tokens and bcrypt password hashing
- Real-time communication using Socket.io
- Media upload functionality with Multer middleware
- Cloud storage integration with Cloudinary
- State management using Redux Toolkit
- RESTful API architecture following MVC pattern
- Responsive design with modern UI/UX principles

The application demonstrates proficiency in modern web development practices,
including component-based architecture, middleware implementation, database 
modeling with MongoDB, and real-time bidirectional event-based communication.

6.1 DATABASE OVERVIEW
---------------------

6.2 COLLECTIONS AND SCHEMAS
----------------------------

6.2.1 USER COLLECTION (users)
-----------------------------
Purpose: Stores user account information and relationships

Schema Structure:
{
  name: String (required)
    - User's full name
  
  userName: String (required, unique)
    - Unique username for identification
  
  email: String (required, unique)
    - Email address for authentication
  
  password: String (required)
    - Bcrypt hashed password
  
  profileImage: String
    - URL to profile picture
  
  bio: String
    - User biography/description
  
  profession: String
    - User's profession or occupation
  
  gender: String
    - User's gender
  
  followers: [ObjectId] → Reference to User
    - Array of user IDs who follow this user
  
  following: [ObjectId] → Reference to User
    - Array of user IDs this user follows
  
  posts: [ObjectId] → Reference to Post
    - Array of post IDs created by user
  
  saved: [ObjectId] → Reference to Post
    - Array of saved/bookmarked post IDs
  
  loops: [ObjectId] → Reference to Loop
    - Array of loop (short video) IDs
  
  story: ObjectId → Reference to Story
    - Reference to user's active story
  
  resetOtp: String
    - OTP for password reset
  
  otpExpires: Date
    - Expiration time for OTP
  
  isOtpVerified: Boolean (default: false)
    - OTP verification status
  
  timestamps: true (createdAt, updatedAt)
}

Indexes:
- userName: Unique index
- email: Unique index

6.2.2 POST COLLECTION (posts)
-----------------------------
Purpose: Stores user posts with media content

Schema Structure:
{
  author: ObjectId → Reference to User (required)
    - ID of the user who created the post
  
  mediaType: String (required)
    - Enum: ["image", "video"]
    - Type of media content
  
  media: String (required)
    - URL to the media file
  
  caption: String
    - Post caption/description
  
  likes: [ObjectId] → Reference to User
    - Array of user IDs who liked the post
  
  comments: [
    {
      author: ObjectId → Reference to User
        - ID of comment author
      message: String
        - Comment text
    }
  ]
    - Array of comment objects
  
  timestamps: true (createdAt, updatedAt)
}

Indexes:
- author: For querying user's posts
- createdAt: For chronological sorting

6.2.3 CONVERSATION COLLECTION (conversations)
--------------------------------------------
Purpose: Manages chat conversations between users

Schema Structure:
{
  participants: [ObjectId] → Reference to User
    - Array of user IDs in the conversation
  
  messages: [ObjectId] → Reference to Message
    - Array of message IDs in this conversation
  
  timestamps: true (createdAt, updatedAt)
}

6.2.4 MESSAGE COLLECTION (messages)
----------------------------------
Purpose: Stores individual chat messages

Schema Structure:
{
  sender: ObjectId → Reference to User
    - ID of the message sender
  
  receiver: ObjectId → Reference to User
    - ID of the message receiver
  
  message: String
    - Message content
  
  timestamps: true (createdAt, updatedAt)
}

6.2.5 LOOP COLLECTION (loops)
----------------------------
Purpose: Stores short-form video content (similar to Reels/TikTok)

Schema Structure:
{
  author: ObjectId → Reference to User
    - ID of the user who created the loop
  
  video: String
    - URL to the video file
  
  caption: String
    - Loop caption/description
  
  likes: [ObjectId] → Reference to User
    - Array of user IDs who liked the loop
  
  comments: [
    {
      author: ObjectId → Reference to User
      message: String
    }
  ]
  
  timestamps: true (createdAt, updatedAt)
}

6.2.6 STORY COLLECTION (stories)
-------------------------------
Purpose: Stores temporary story content (24-hour duration)

Schema Structure:
{
  author: ObjectId → Reference to User
    - ID of the user who created the story
  
  mediaType: String
    - Enum: ["image", "video"]
  
  media: String
    - URL to the media file
  
  expiresAt: Date
    - Expiration timestamp (24 hours)
  
  views: [ObjectId] → Reference to User
    - Array of user IDs who viewed the story
  
  timestamps: true (createdAt, updatedAt)
}

6.2.7 NOTIFICATION COLLECTION (notifications)
-------------------------------------------
Purpose: Stores user notifications

Schema Structure:
{
  recipient: ObjectId → Reference to User
    - ID of the user receiving the notification
  
  sender: ObjectId → Reference to User
    - ID of the user who triggered the notification
  
  type: String
    - Enum: ["like", "comment", "follow", "message"]
  
  post: ObjectId → Reference to Post
    - Related post ID (if applicable)
  
  message: String
    - Notification message text
  
  read: Boolean (default: false)
    - Read status
  
  timestamps: true (createdAt, updatedAt)
}

6.3 RELATIONSHIP DIAGRAM
------------------------
User ←→ User (followers/following - Many-to-Many)
User → Post (One-to-Many)
User → Loop (One-to-Many)
User → Story (One-to-One)

Post → Comment (Embedded Array)

Conversation → Message (One-to-Many)

7.2 AUTHENTICATION SYSTEM
-------------------------
Implementation: auth.controllers.js

7.2.1 User Registration (signUp):
- Accepts: name, userName, email, password, gender
- Process:
  1. Validates input data
  2. Checks for existing username/email
  3. Hashes password using bcrypt
  4. Creates new user document
  5. Generates JWT token
  6. Sets HTTP-only cookie
  7. Returns user data

7.2.2 User Login (signIn):
- Accepts: email/username, password
- Process:
  1. Finds user by email or username
  2. Verifies password with bcrypt compare
  3. Generates JWT token
  4. Sets authentication cookie
  5. Returns user data

7.2.3 User Logout (signOut):
- Clears authentication cookie
- Returns success message

7.2.4 Session Validation (me):
- Protected route using isAuth middleware
- Returns current user data if authenticated

7.3 AUTHENTICATION MIDDLEWARE
-----------------------------
Implementation: middlewares/isAuth.js

Process:
1. Extracts JWT token from cookies
2. Verifies token using secret key
3. Retrieves user from database
4. Attaches user to request object
5. Allows request to proceed or returns 401

7.4 POST MANAGEMENT SYSTEM
--------------------------
Implementation: post.controllers.js

7.4.1 Upload Post (uploadPost):
- Uses multer middleware for file handling
- Accepts: media file, caption, mediaType
- Process:
  1. Validates file upload
  2. Stores file locally in public directory
  3. Creates post document in database
  4. Updates user's posts array
  5. Returns created post data

7.4.2 Get All Posts (getAllPosts):
- Retrieves posts from database
- Populates author information
- Sorts by creation date (newest first)
- Returns array of posts

7.4.3 Like Post (like):
- Toggles like status for authenticated user
- Updates likes array in post document
- Returns updated like status

7.4.4 Update Post (updatePost):
- Allows author to edit post
- Supports media replacement
- Updates caption and/or media
- Returns updated post

7.4.5 Delete Post (deletePost):
- Verifies author authorization
- Removes post document
- Updates user's posts array
- Deletes associated media file
- Returns success message

7.4.6 Comment Management:
- Add Comment: Creates new comment on post
- Update Comment: Edits existing comment (author only)
- Delete Comment: Removes comment (author only)

7.5 FILE UPLOAD MIDDLEWARE
--------------------------
Implementation: middlewares/multer.js

Configuration:
- Storage: Disk storage in public directory
- Destination: public/ folder
- Filename: Unique name with original extension
- File Filter: Validates file types
- Size Limit: Configurable maximum size

7.6 SOCKET.IO IMPLEMENTATION
----------------------------
Implementation: socket.js

Features:
1. WebSocket Server Setup:
   - Creates HTTP server with Express app
   - Initializes Socket.io with CORS configuration
   - Allows connections from frontend origin

2. User Socket Mapping:
   - userSocketMap: Object storing userId → socketId
   - getSocketId(): Helper function to retrieve socket ID

3. Connection Handling:
   - Extracts userId from handshake query
   - Stores socket ID in map
   - Broadcasts online users list

4. Disconnection Handling:
   - Removes user from socket map
   - Updates online users list
   - Notifies all clients

5. Real-time Features:
   - Online user status
   - Instant notifications
   - Real-time messaging
   - Live updates
